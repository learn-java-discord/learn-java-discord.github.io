About this FAQ

This FAQ was created by the Learn Java Discord Channel to help anyone new with Java and those looking for guidance. There are many challenges with learning the Java ecosystem and the JVM stack, but hopefully this guide gives a coherent introduction to the language.

Where do I start?

This depends on your goals as a Java programmer. The sections below gives some advice depending on whether your priority is enterprise, mobile, or web development. Enterprise development is particularly important if you are simply looking for work. Regardless, starting with vanilla Java is a given.

Installing intelliJ IDEA (or some equivalent IDE like Eclipse or Netbeans) is a the the most recommended first step. The community edition is available for free right here and is one of the more popular and well received IDEs for Java development.

Starting Development

Now that you have installed an IDE, it is time to learn the basics of Java. If Java is your first programming language, then the two "Object Oriented Programming With Java" courses by MOOC are a well structured place to start. These courses are self-instructed and thus can be taken at your own pace. You can find them here.

Furthermore, here are a few lists of books and publications that are clasically regarded as the best way to approach Java as a beginner to programming:

  • Head First Java by O'Reilly
  • Thinking In Java (4th Edition) by Bruce Eckel

  • Absolute Java (6th Edition) by Walter Savitch and Kenrick Mock

Learning about general programming concepts that aren't specific to Java is a good plan. Most people you ask about where to start will often tell you to begin with design patterns, data structures, algorithms etc. This might be less hands on and a little drier than you want, but learning them sooner can save sweat and tears later. Below are some recommended books about programming in general.

  • Code Complete (2nd Edition) by Steve McConnell
  • Head First Design Patterns by Eric Freeman, Bert Bates, and Kathy Sierra
  • The Pragmatic Programmer by Andrew Hunt and David Thomas
  • Clean Code by Robert C. Martin
  • Introduction to Algorithms by Thomas M. Cormen, Charles E. Leiserson, Ronald L. Rivest, and Clifford Stein
